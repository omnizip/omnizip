== Performance Notes

**Omnizip Performance:**

* 10-60x slower than native C implementations
* Excellent for Ruby-native environments
* Acceptable for moderate file sizes (<100MB)
* Memory-efficient streaming support
* No compilation or native dependencies

See link:performance-tuning.html[Performance Tuning] for detailed benchmarks.

== Use Case Decision Matrix

[cols="30,70",options="header"]
|===
|Choose This Tool |When You Need

|**Omnizip**
|Pure Ruby, 7-Zip support, advanced algorithms, rubyzip compatibility, no dependencies

|**rubyzip**
|Simple ZIP operations, mature Ruby library, existing codebase integration

|**7-Zip**
|Maximum compression, command-line usage, native performance, multi-format support

|**WinRAR**
|RAR write support, Windows GUI, commercial support, recovery features

|**PeaZip**
|Open-source GUI, many formats, file management features, batch operations

|**libarchive**
|C library integration, many formats, streaming support, production systems

|**Info-ZIP**
|Traditional ZIP tools, shell scripts, basic operations, legacy systems

|===

== Summary

Omnizip excels in:

* **Pure Ruby implementation** - No compilation or dependencies
* **Advanced compression** - LZMA2, PPMd, Deflate64 support
* **7-Zip compatibility** - Full .7z format support in Ruby
* **Rubyzip compatibility** - Drop-in replacement with enhancements
* **API quality** - Clean, well-documented Ruby API
* **Portability** - Runs anywhere Ruby runs

Consider alternatives for:

* **Maximum performance** - Use native tools (7-Zip, WinRAR)
* **Simple ZIP-only needs** - rubyzip may be sufficient
* **RAR write support** - WinRAR required
* **GUI operations** - PeaZip or native tools
